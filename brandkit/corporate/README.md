# Brand Kit Workflow

This folder is a lightweight brand reference that other workflows can read before generating user-facing content, documentation, UI copy, visuals, or diagrams.

It is intentionally generic. Replace the placeholder guidance with your own brand decisions as your project matures.

## How to Use This Folder

1. Start with [01-primary-brand.md](01-primary-brand.md) to define the brand foundation.
2. Use [02-voice-and-tone.md](02-voice-and-tone.md) before writing content.
3. Use [03-writing-guide.md](03-writing-guide.md) for style, formatting, and terminology.
4. Use [04-colors.md](04-colors.md) for visual design decisions.
5. Use [05-icons.md](05-icons.md) for icon selection and usage.
6. Use [06-logo.md](06-logo.md) once a logo or wordmark exists.

## Location Note

This folder lives at `brandkit/corporate/` within the Brand Kit workspace. Other sub-brands (personal, product, etc.) live as siblings under `brandkit/`. To start a new brand, copy the `_template/` folder and follow `SETUP.md`.

## Editing Rules

- Remove any placeholder text that does not match your project.
- Keep the files short enough for an AI tool to load as context.
- Prefer clear decisions over long explanations.
- Do not include private, proprietary, or client-specific brand assets in this public version.
- When a workflow needs brand guidance, point it to this folder instead of duplicating brand rules.

## Editing with AI

- You can always ask AI to update with your info. 
- You can always start from cratch and tell AI what you ant or feed it your brand requirements if you have them
- if you have social media posts you can give it those to fine tune to your voice, tone, etc.
