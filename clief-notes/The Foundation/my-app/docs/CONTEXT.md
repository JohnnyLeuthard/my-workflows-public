# Documentation

User-facing and developer-facing documentation.

## What's here
- /api — API reference documentation
- /guides — How-to guides and tutorials
- /changelog — Version history and breaking changes

## How to work here
- Update /api docs when endpoints or interfaces change
- Add a /changelog entry for every release or breaking change
- Guides should be task-oriented — "How to [do X]" not "About [concept]"
