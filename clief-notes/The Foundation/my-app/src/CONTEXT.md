# Source Code

This is the application codebase.

## What's here
- /components — [UI components / modules / handlers]
- /services — [Business logic, API clients, integrations]
- /utils — [Shared helpers and utilities]
- /tests — [Test files]

## How to work here
- Check /planning/specs for requirements before building new features
- Keep components focused — one responsibility per file
- Shared logic goes in /utils, not duplicated across components
- Every new feature or fix should have corresponding tests in /tests
