Yours is one of the few systems here that has verifiably been run in anger: dated sends in the Current State log, a pattern-guessed address that didn't bounce, policy changes I can trace propagating into the compliance checklist and the card template, and known issues listed the way an operator lists them. The human gate is also the best-reasoned in the field — held not as a vibe but because a sent cold email is reputation-bearing and legally consequential, and that setting holds regardless of trust level. The Commons roundtable — a plain-text log as the inter-agent protocol, read first, post one entry last — is an idea worth extracting from this build and shipping on its own.

The push is the public cut. It ships the private repo's gitignore header ("NEVER push to a public repo"), an examples file whose own warning says it isn't ready for public DSGVO use while still naming the org, a dozen dangling references (clients/daa, the werkzeuge tool called runnable but absent, /wrap-up), and typos on the front door. The em-dash rule is instruction-only when you've already named the deterministic hook that fixes it. An afternoon's integrity pass and this reads like what it is. One correction for the bausteine explainer: ICM isn't out of Edinburgh — check the paper's actual affiliation before that page travels.
