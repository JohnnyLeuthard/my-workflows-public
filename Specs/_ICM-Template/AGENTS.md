# This Workspace Is Not Set Up Yet (Seed Mode)

**You are reading the Layer 0 file of an ICM workspace that has not been built out.** This
folder started life as a copy of the **ICM Workspace Template** and has not yet been through
setup. Everything here exists to turn this folder into a real, working ICM workspace — and then
get out of the way.

Mind the two levels:

- **The template** (`_ICM-Template/`) is a copy-and-rename seed. It never does work itself.
- **This copy** — renamed to a workspace name and set up — becomes a real workspace with its
  own identity, router, and stages. Setup **rewrites this file and `CONTEXT.md`** into that
  workspace's real front door; the seed-mode text you are reading now will be gone.

## Protocol

1. Read `CONTEXT.md` in this folder next — it routes every request this folder can currently
   handle (setup, explanation, design). Do not guess at work this workspace "should" do; until
   setup completes, there is no such work.
2. Setup is owned by `_setup/AGENTS.md` and follows ICM itself: kickoff questions → optional
   interview → a reviewable, resumable build plan → the build → a mandatory audit. State lives
   in files under `_setup/`, never in a session's memory.
3. The human is in charge at every step: they can stop the interview at any question, edit the
   build plan directly, perform any build step by hand, or take over entirely. Never fight
   that; work from the files as they stand.

For humans: start with `README.md` (the full step-by-step walkthrough) and `NEXT-STEPS.md`
(what to do right now). Agents never route through those files.
