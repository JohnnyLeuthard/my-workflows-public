## Description

Contains Safe-level events that are generated by the Vault, CyberArk components (CPM, PSM, PVWA), and external integrations. Events include notifications, alerts, and system-generated messages associated with specific Safes.

---

## Columns (Out-of-Box)

| # | Column | DB Column | Description |
|---|--------|-----------|-------------|
| 1 | Safe | CAESafe | Name of the Safe the event is associated with |
| 2 | EventID | CAEEventID | Unique event identifier |
| 3 | SourceID | CAESourceID | Source of the event (decode via CATextCodes Type 10) |
| 4 | EventTypeID | CAEEventTypeID | Type of event (decode via CATextCodes Type 11) |
| 5 | ClientID | CAEClientID | Client application identifier |
| 6 | User | CAEUser | User associated with the event |
| 7 | Agent | CAEAgent | Agent/component that generated the event |
| 8 | FromIP | CAEFromIP | IP address of the source |
| 9 | Version | CAEVersion | Version of the client/component |
| 10 | CreationDate | CAECreationDate | Date/time the event was created |
| 11 | ExpirationDate | CAEExpirationDate | Date/time the event expires |
| 12 | EventVersion | CAEEventVersion | Version of the event schema |
| 13 | Data | CAEData | Event payload data (free-text, may contain structured data) |
| 14 | VaultID | CAEVaultID | Vault instance identifier |

## Usage Notes

- Events are joined to Safes by **name** (`CAESafe`), not by ID — be aware of Safe rename implications
- `CAEData` is a free-text field (nvarchar(8000)) that may contain XML, JSON, or delimited data depending on the event source
- Events are distinct from the audit log (`CALog`) — events are notifications/alerts, the log is user activity
- CPM-generated events include password change results, verification results, and reconciliation outcomes
