# CCP Stage 02 Router: Request Catalog

## Use This Stage For

- Defining request item names and descriptions.
- Separating required, optional, and endpoint-specific selectors.
- Mapping user-facing labels to vendor parameter names after vendor docs are reviewed.
- Producing `output/request_catalog.md`.

## Required References

Read `../../references/_INDEX.md`, then `../../references/vendor_docs.md`.

## Stop Rule

After producing or updating the request catalog artifact, stop and wait for human approval before URI building.
