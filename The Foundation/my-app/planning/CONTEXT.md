# Planning

This is where features are designed before they're built.

## What's here
- /specs — Feature specifications and requirements
- /architecture — System design docs, diagrams, data models
- /decisions — Architecture Decision Records (ADRs)

## How to work here
- New features start as a spec in /specs before any code is written
- Architecture changes get an ADR in /decisions
- Reference existing specs when modifying related features in /src
